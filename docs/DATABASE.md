# 🗄️ AXIOM — Database Documentation

> **Engine:** PostgreSQL 16  
> **Local:** H2 in-memory (auto-created, no setup needed)  
> **Production:** PostgreSQL inside Docker on Hostinger VPS

---

## 📐 Architecture Overview

```
┌─────────────────────────────────────────────────────────────┐
│                    PostgreSQL (port 5432)                    │
├──────────────────┬──────────────────┬───────────────────────┤
│   axiom_users    │  axiom_analysis  │    axiom_stocks        │
│  (user-service)  │(analysis-service)│ (stock-data-service)   │
│                  │                  │                        │
│  • users         │ • stock_analyses │  (no persistent tables │
│  • watchlists    │                  │   — live API data only) │
│  • watchlist_    │                  │                        │
│    tickers       │                  │                        │
└──────────────────┴──────────────────┴───────────────────────┘
```

## 🏛️ Database: `axiom_users`

Managed by **user-service** (port 8081)

---

### Table: `users`

Stores all registered user accounts.

| Column           | Type        | Nullable | Default     | Description                          |
|------------------|-------------|----------|-------------|--------------------------------------|
| `id`             | UUID        | NO       | auto-gen    | Primary key                          |
| `email`          | VARCHAR     | NO       | —           | Unique login email                   |
| `username`       | VARCHAR     | YES      | —           | Unique username (e.g. `raghu`)       |
| `password_hash`  | VARCHAR     | NO       | —           | BCrypt hashed password               |
| `first_name`     | VARCHAR     | YES      | —           | First name                           |
| `last_name`      | VARCHAR     | YES      | —           | Last name                            |
| `role`           | VARCHAR     | NO       | `USER`      | `USER` / `PREMIUM` / `ADMIN`         |
| `tier`           | VARCHAR     | NO       | `FREE`      | `FREE` / `PRO` / `ENTERPRISE`        |
| `email_verified` | BOOLEAN     | NO       | `false`     | Email verification status            |
| `fcm_token`      | VARCHAR     | YES      | —           | Firebase push notification token     |
| `default_horizon`| VARCHAR     | YES      | —           | User's preferred trading horizon     |
| `created_at`     | TIMESTAMPTZ | YES      | auto-set    | Account creation time                |
| `updated_at`     | TIMESTAMPTZ | YES      | auto-set    | Last update time                     |

**Constraints:**
- `UNIQUE(email)`
- `UNIQUE(username)`

**Pre-seeded accounts (via DataInitializer on startup):**

| username | email            | password     | role    | tier         |
|----------|------------------|--------------|---------|--------------|
| `raghu`  | raghu@axiom.ai   | `bigsky3016` | `ADMIN` | `ENTERPRISE` |
| `demo`   | demo@axiom.ai    | `Demo1234!`  | `USER`  | `FREE`       |

---

### Table: `watchlists`

Stores named watchlists per user.

| Column    | Type    | Nullable | Description              |
|-----------|---------|----------|--------------------------|
| `id`      | UUID    | NO       | Primary key              |
| `user_id` | UUID    | NO       | FK → `users.id`          |
| `name`    | VARCHAR | NO       | Watchlist name           |

---

### Table: `watchlist_tickers`

Stores individual ticker symbols in each watchlist (one row per ticker).

| Column         | Type    | Nullable | Description              |
|----------------|---------|----------|--------------------------|
| `watchlist_id` | UUID    | NO       | FK → `watchlists.id`     |
| `ticker`       | VARCHAR | NO       | Stock ticker (e.g. AAPL) |

## 📊 Database: `axiom_analysis`

Managed by **analysis-service** (port 8083)

Stores every AI analysis result generated by a user.

| `user_id`           | UUID        | YES      | Which user ran this analysis          |
| `executive_summary` | TEXT        | YES      | AI executive summary section          |
| `market_pulse`      | TEXT        | YES      | Market context section                |
| `technical_analysis`| TEXT        | YES      | Technical indicators section          |
| `support_resistance`| TEXT        | YES      | S/R levels section                    |
| `fundamentals`      | TEXT        | YES      | Fundamentals section                  |
| `entry_exit_signals`| TEXT        | YES      | Entry/exit signals section            |
| `bull_bear_scorecard`| TEXT       | YES      | Bull/Bear score breakdown             |
| `risk_factors`      | TEXT        | YES      | Risk factors section                  |
| `trade_plan`        | TEXT        | YES      | Full trade plan section               |
| `overall_score`     | INTEGER     | YES      | 0–100 composite score                 |
| `verdict`           | VARCHAR     | YES      | `STRONG_BULL`/`MILD_BULL`/`NEUTRAL`/`MILD_BEAR`/`STRONG_BEAR` |
| `entry_low`         | DECIMAL     | YES      | Lower entry price bound               |
| `entry_high`        | DECIMAL     | YES      | Upper entry price bound               |
| `stop_loss`         | DECIMAL     | YES      | Stop loss price                       |
| `target1`           | DECIMAL     | YES      | Profit target 1                       |
| `target2`           | DECIMAL     | YES      | Profit target 2                       |
| `target3`           | DECIMAL     | YES      | Profit target 3                       |
| `risk_reward`       | DECIMAL     | YES      | Risk/reward ratio (e.g. 2.8 = 1:2.8) |
| `custom_analysis`   | TEXT        | YES      | Result of custom prompt section       |
| `created_at`        | TIMESTAMPTZ | NO       | When analysis was run                 |
| `expires_at`        | TIMESTAMPTZ | YES      | Cache expiry (30 min TTL)             |

## 🐳 Database: `axiom_stocks`

Managed by **stock-data-service** (port 8082)

> **No persistent tables.** All stock data (quotes, OHLCV, technicals, news) is fetched live from Yahoo Finance / Finnhub / Alpha Vantage APIs and returned directly. Redis is used for short-term caching only.

## 🖥️ Accessing the Database on Hostinger VPS

### Step 1 — Get your VPS IP from Hostinger Panel

1. Log in → [hpanel.hostinger.com](https://hpanel.hostinger.com)
2. Click **VPS** in the left menu
3. Click **Manage** on your VPS
4. Copy the **IPv4 address** shown on the overview page

### Step 2 — SSH into your server
```bash
ssh root@YOUR_HOSTINGER_IP
```
> Default SSH port is **22**. If you changed it in Hostinger panel, use: `ssh root@YOUR_IP -p YOUR_PORT`

### Step 3 — Open psql inside the Docker container
```bash
# Connect to axiom_users
docker exec -it $(docker ps -qf "name=postgres") psql -U axiom -d axiom_users

# Connect to axiom_analysis
docker exec -it $(docker ps -qf "name=postgres") psql -U axiom -d axiom_analysis
```

### Step 3 — Common queries

```sql
-- List all databases
\l

-- List all tables in current DB
\dt

-- Show all registered users
SELECT id, email, username, first_name, last_name, role, tier, email_verified, created_at
FROM users
ORDER BY created_at;

-- Check admin user
SELECT * FROM users WHERE role = 'ADMIN';

-- Count total analyses run
\c axiom_analysis
SELECT COUNT(*) as total_analyses FROM stock_analyses;

-- Top 10 most analysed tickers
SELECT ticker, COUNT(*) as count
FROM stock_analyses
GROUP BY ticker
ORDER BY count DESC
LIMIT 10;

-- Latest analyses
SELECT ticker, horizon, overall_score, verdict, created_at
FROM stock_analyses
ORDER BY created_at DESC
LIMIT 20;

-- User watchlists
\c axiom_users
SELECT w.name, wt.ticker
FROM watchlists w
JOIN watchlist_tickers wt ON w.id = wt.watchlist_id
ORDER BY w.user_id, w.name;

-- Exit
\q
```

## 💻 Remote Access from Windows (GUI)

### Option A — DBeaver / pgAdmin via SSH Tunnel (recommended)

**Step 1:** Find your IP in [hpanel.hostinger.com](https://hpanel.hostinger.com) → VPS → Manage

**Step 2:** Open PowerShell and create SSH tunnel:
```powershell
ssh -L 5433:localhost:5432 root@YOUR_HOSTINGER_IP -N
```
> Keep this terminal open — it tunnels Hostinger's port 5432 → your local port 5433.

**Step 3:** In DBeaver → New Connection → PostgreSQL:

| Field    | Value       |
|----------|-------------|
| Host     | `localhost` |
| Port     | `5433`      |
| Database | `axiom_users` |
| Username | `axiom`     |
| Password | *(from `/opt/axiom/.env`)* |

### Option B — psql from Windows terminal
```powershell
# With SSH tunnel running on port 5433
psql -h localhost -p 5433 -U axiom -d axiom_users
```

### Get the password from .env on VPS
```bash
ssh root@YOUR_HOSTINGER_IP "cat /opt/axiom/.env | grep DB_PASSWORD"
```

## 🔑 Authentication Flow (users table)

```
User submits login (email OR username + password)
        │
        ▼
AuthService.login()
        │
        ├─ findByEmailOrUsername(input, input)   ← searches users table
        │
        ├─ BCrypt.matches(rawPassword, passwordHash)
        │
        ├─ Generate JWT accessToken (15 min expiry)
        │
        ├─ Generate refreshToken (UUID, 7 day expiry)
        │   └─ stored in ConcurrentHashMap (in-memory, not DB)
        │
        └─ Return { accessToken, refreshToken, user: UserDto }
```

> ⚠️ **Note:** Refresh tokens are stored **in-memory** (not in the DB). They are lost on service restart. For production, move to Redis or a `refresh_tokens` table.

## 🌱 Seed Data

The `DataInitializer` bean runs on every startup and creates these accounts if they don't already exist:

```java
// backend/user-service/src/main/java/com/axiom/user/config/DataInitializer.java
seedAdmin("raghu@axiom.ai", "raghu",  "bigsky3016", "Raghu", "Admin");  // ADMIN / ENTERPRISE
seedAdmin("demo@axiom.ai",  "demo",   "Demo1234!",  "Demo",  "User");   // USER  / FREE
```

## 🔄 DDL Strategy

| Environment | `ddl-auto` value | Behaviour                              |
|-------------|------------------|----------------------------------------|
| Local       | `create-drop`    | Tables created on start, dropped on stop |
| Docker prod | `update`         | Tables created/altered, data preserved |

> Tables are **auto-managed by Hibernate** — no manual SQL migrations needed for schema changes. Just change the Java entity and restart.
